## 二、方案A：自己电脑作为本地服务器（演示/开发用）

适合：笔记本演示、本机调试

### 步骤

1. 解压 zip 到任意目录，例如 `D:\rfmatrix-demo`

2. 打开 **PowerShell** 或 **命令提示符**，进入项目目录：
   ```powershell
   cd D:\rfmatrix-demo
   ```

3. 安装依赖（首次使用，需联网，约 1~2 分钟）：
   ```powershell
   npm install
   ```

4. 启动本地服务器：
   ```powershell
   npm run dev
   ```
   看到如下输出即表示启动成功：
   ```
   ➜  Local:   http://localhost:5173/
   ```

5. 打开浏览器，访问 `http://localhost:5173`

### 让局域网内其他人也能访问（投屏/同网演示）

```powershell
npm run dev -- --host
```

启动后会显示本机局域网 IP，例如：
```
➜  Network: http://192.168.1.100:5173/
```
同一 WiFi 下的其他设备用该地址即可访问。

### 关闭服务器

在终端窗口按 `Ctrl + C`，或直接关闭终端窗口。

## 三、方案B：部署到实际服务器（Linux 生产环境）

适合：部署到云服务器、内网服务器，供多人长期访问。

> **关键要求**：服务器必须返回以下两个 HTTP 响应头，否则 sql.js WASM 无法运行：
> ```
> Cross-Origin-Opener-Policy: same-origin
> Cross-Origin-Embedder-Policy: require-corp
> ```

### 步骤 1：构建产物

在开发机器（或服务器上）执行：

```bash
npm install
npm run build
```

生成 `dist/` 目录，内含所有静态文件。

---

### 步骤 2A：用 Node.js 简易服务器托管（推荐，最简单）

在服务器上，在项目根目录新建 `server.js`：

```js
const http = require('http');
const fs   = require('fs');
const path = require('path');

const DIST = path.join(__dirname, 'dist');
const PORT = 3000;

const MIME = {
  '.html': 'text/html',
  '.js':   'application/javascript',
  '.css':  'text/css',
  '.svg':  'image/svg+xml',
  '.png':  'image/png',
  '.ico':  'image/x-icon',
  '.sql':  'text/plain',
  '.wasm': 'application/wasm',
  '.json': 'application/json',
};

http.createServer((req, res) => {
  // 必须的安全响应头（sql.js WASM 要求）
  res.setHeader('Cross-Origin-Opener-Policy',   'same-origin');
  res.setHeader('Cross-Origin-Embedder-Policy', 'require-corp');

  let filePath = path.join(DIST, req.url === '/' ? '/index.html' : req.url);

  // 路径不存在时回退到 index.html（SPA 路由）
  if (!fs.existsSync(filePath) || fs.statSync(filePath).isDirectory()) {
    filePath = path.join(DIST, 'index.html');
  }

  const ext  = path.extname(filePath);
  const mime = MIME[ext] || 'application/octet-stream';

  fs.readFile(filePath, (err, data) => {
    if (err) { res.writeHead(404); res.end('Not found'); return; }
    res.writeHead(200, { 'Content-Type': mime });
    res.end(data);
  });
}).listen(PORT, () => console.log(`Server running at http://localhost:${PORT}`));
```

启动：
```bash
node server.js
```

后台持久运行（使用 pm2）：
```bash
npm install -g pm2
pm2 start server.js --name rfmatrix
pm2 save
pm2 startup   # 设置开机自启
```

---

### 步骤 2B：用 Nginx 托管

在 Nginx 配置文件（如 `/etc/nginx/sites-available/rfmatrix`）中添加：

```nginx
server {
    listen 80;
    server_name 你的域名或IP;

    root /var/www/rfmatrix/dist;
    index index.html;

    # 必须的响应头
    add_header Cross-Origin-Opener-Policy   "same-origin"   always;
    add_header Cross-Origin-Embedder-Policy "require-corp"  always;

    location / {
        try_files $uri $uri/ /index.html;  # SPA 路由支持
    }

    # wasm 文件 MIME 类型
    location ~* \.wasm$ {
        add_header Content-Type application/wasm;
        add_header Cross-Origin-Opener-Policy   "same-origin"   always;
        add_header Cross-Origin-Embedder-Policy "require-corp"  always;
    }
}
```

启用并重启：
```bash
ln -s /etc/nginx/sites-available/rfmatrix /etc/nginx/sites-enabled/
nginx -t       # 检查配置
systemctl reload nginx
```

## 四、腾讯云完整部署流程（互联网公开访问）

### 4.1 购买服务器

推荐选择**轻量应用服务器（Lighthouse）**，比 CVM 操作更简单：

1. 登录 [腾讯云控制台](https://console.cloud.tencent.com/)
2. 搜索「轻量应用服务器」→ 新建实例
3. 镜像选择：**Ubuntu 22.04 LTS**（推荐）
4. 配置：1核2G 内存即可满足演示需求
5. 设置登录密码（记住，后面 SSH 会用到）
6. 购买完成后记录服务器的**公网 IP**

---

### 4.2 连接服务器

**方式一：网页终端（最简单，无需任何软件）**

在轻量应用服务器控制台 → 选中实例 → 点击「登录」→ 选择「网页终端」

**方式二：本机 PowerShell SSH**

```powershell
ssh root@你的公网IP
# 输入购买时设置的密码
```

---

### 4.3 服务器环境准备

连接成功后，在终端依次执行：

```bash
# 更新包列表
apt update

# 安装 Node.js 18（通过 NodeSource 官方源）
curl -fsSL https://deb.nodesource.com/setup_18.x | bash -
apt install -y nodejs

# 验证安装
node -v   # 应输出 v18.x.x
npm -v

# 安装 pm2（进程守护，让服务器关闭 SSH 后程序继续运行）
npm install -g pm2
```

---

### 4.4 上传项目文件

**方式一：WinSCP（图形界面，推荐新手）**

1. 下载安装 [WinSCP](https://winscp.net/)
2. 新建连接：协议 SFTP，主机名填公网 IP，用户名 `root`，密码同上
3. 连接后将本机 `rfmatrix-demo/` 文件夹（**不含 node_modules 和 dist**）拖拽到服务器的 `/root/` 目录下

**方式二：PowerShell SCP 命令**

```powershell
# 先在本机打包（排除 node_modules 和 dist）
# 手动压缩或用命令，然后传到服务器：
scp C:\Users\85256\Desktop\rfmatrix-demo.zip root@你的公网IP:/root/
```

然后在服务器上解压：
```bash
apt install -y unzip
unzip /root/rfmatrix-demo.zip -d /root/rfmatrix-demo
```

---

### 4.5 安装依赖并构建

```bash
# 进入项目目录
cd /root/rfmatrix-demo

# 安装依赖
npm install

# 构建生产产物
npm run build
# 成功后会出现 dist/ 目录
```

---

### 4.6 启动服务

确认项目根目录有 `server.js`（见本文第三节），然后：

```bash
# 用 pm2 启动（后台持久运行）
pm2 start server.js --name rfmatrix

# 查看运行状态
pm2 status

# 设置开机自启
pm2 save
pm2 startup
# 按照 pm2 输出的提示再执行一条命令
```

服务启动后默认监听 **3000 端口**。

---

### 4.7 开放防火墙端口（重要）

腾讯云默认只开放 22 端口，必须手动放行：

1. 进入轻量应用服务器控制台 → 选中实例 → 「防火墙」选项卡
2. 点击「添加规则」：
   - 应用类型：自定义
   - 协议：TCP
   - 端口：`3000`
   - 来源：`0.0.0.0/0`（所有人可访问）
3. 保存

---

### 4.8 验证访问

浏览器打开：

```
http://你的公网IP:3000
```

能看到系统登录页即部署成功。

---

### 4.9 日常运维命令

```bash
pm2 status            # 查看服务状态
pm2 logs rfmatrix     # 查看实时日志
pm2 restart rfmatrix  # 重启服务
pm2 stop rfmatrix     # 停止服务

# 更新代码后重新部署
cd /root/rfmatrix-demo
npm run build
pm2 restart rfmatrix
```

---

### 4.10 可选：绑定域名 + HTTPS

若有已备案域名，在腾讯云 DNS 解析中添加 A 记录指向服务器 IP，并申请免费 SSL 证书后配置 Nginx 反向代理，可通过 `https://你的域名` 访问（此步骤按需操作）。

## 五、免费部署方案（Netlify）

适合：快速上线演示，无需购买服务器，**完全免费**。

> Netlify 免费套餐：100GB 流量/月、自动 HTTPS、自定义域名，满足演示需求。

---

### 5.1 准备工作：添加响应头配置

Netlify 通过项目 `public/` 目录下的 `_headers` 文件来设置 HTTP 响应头。

在 `public/_headers` 文件中写入（该文件已存在则追加）：

```
/*
  Cross-Origin-Opener-Policy: same-origin
  Cross-Origin-Embedder-Policy: require-corp
```

> 此文件构建时会被自动复制到 `dist/` 中，Netlify 会读取它。

---

### 5.2 构建产物

在本机执行：

```powershell
cd C:\Users\85256\Desktop\前端DEMO\rfmatrix-demo
npm run build
```

生成 `dist/` 目录。

---

### 5.3 部署到 Netlify（拖拽上传，2 分钟完成）

1. 打开 [https://app.netlify.com/drop](https://app.netlify.com/drop)（无需注册账号）
2. 将本机的 `dist/` 文件夹**直接拖拽**到页面中央的上传区域
3. 等待上传完成（约 10~30 秒）
4. Netlify 自动分配一个公网地址，如：`https://random-name-123.netlify.app`
5. 点击该链接即可访问

> 若需要固定网址或自定义域名，注册 Netlify 账号后可修改站点名称。

---

### 5.4 更新内容

重新 `npm run build` 后，重复拖拽 `dist/` 文件夹到同一个站点页面即可覆盖更新。

---

### 5.5 与腾讯云方案对比

| | Netlify 免费 | 腾讯云轻量服务器 |
|---|---|---|
| 费用 | 免费 | 约 50~100 元/月 |
| 部署难度 | 极简（拖拽）| 需要 Linux 操作 |
| 访问速度（国内）| 一般（海外节点）| 快（国内节点）|
| 数据持久性 | 每个访客独立 localStorage | 同上 |
| 适合场景 | 临时演示、对外分享链接 | 长期运行、内网部署 |

## 六、常见问题

| 现象 | 原因 | 解决 |
|---|---|---|
| 页面空白 / 控制台报 `SharedArrayBuffer` 错误 | 缺少 COOP/COEP 响应头 | 检查服务器配置是否正确返回这两个头 |
| 数据库加载失败 | sql.js CDN 不可访问 | 确认服务器能访问 `cdnjs.cloudflare.com` |
| 刷新页面后数据丢失 | 换了浏览器或清了缓存 | 数据存在 localStorage，同浏览器同域名下持久保存 |
| 局域网其他人无法访问 | 本机防火墙拦截 | 在 Windows 防火墙中放行 5173 端口（或 3000 端口）|
